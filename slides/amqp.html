Протокол AMQP

Протокол для обмена сообщений между компонентами системы
  • Работает по принципу FIFO (но это не точно)
  • Быстрый, не зависит от местонахождения и количества (один сервак, тысячи их)
  • Состоит из 4 частей (продюсер, шина, консюмер, сообщение-фрейм)
  • Есть куча библиотек для разных ЯП
  • Можно строить Peer-to-Peer, а можно отсылать тысячам консюмеров
